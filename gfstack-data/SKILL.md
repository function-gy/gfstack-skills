# gfstack-data

## 1. Entity

TRIGGER: entity, ORM entity, table mapping, orm tag, gtime.Time, soft delete

Location: `internal/model/entity/{table}.go`

Generated by `gf gen dao`. 1:1 mapping to database tables. **NEVER manually create.**

```go
type Foo struct {
    Id        int64       `json:"id"         orm:"id"          description:""`
    Name      string      `json:"name"       orm:"name"        description:"名称"`
    Status    string      `json:"status"     orm:"status"      description:"状态"`
    CreatedAt *gtime.Time `json:"createdAt"  orm:"created_at"  description:"创建时间"`
    UpdatedAt *gtime.Time `json:"updatedAt"  orm:"updated_at"  description:"更新时间"`
    DeletedAt *gtime.Time `json:"deletedAt"  orm:"deleted_at"  description:"删除时间"`
}
```

**Key rules:**
- JSON field: camelCase; ORM field: snake_case
- Time fields: `*gtime.Time`
- Soft delete: `deleted_at` field — framework auto-filters, NEVER add `WhereNull(deleted_at)` manually
- `created_at` / `updated_at`: framework auto-maintains, NEVER set manually
- Entity auto-generated by `gf gen dao` uses `description` tag (GoFrame equivalent of `dc`); DO NOT change to `dc` manually — they will be overwritten on regeneration

## 2. DO (Data Objects)

TRIGGER: DO, data object, ORM write

Location: `internal/model/do/{table}.go`

Generated. Used with `Data()` for Insert/Update/Save. All fields are `interface{}` — unset fields are `nil` and auto-ignored by ORM.

## 4. Form Base Types

TRIGGER: PageReq, PageRes, pagination, Pack, pageSize

Location: `internal/model/input/form/`

```go
type PageReq struct {
    Page       int  `json:"page"       dc:"页码"     d:"1"  v:"page@min:1"`
    PerPage    int  `json:"pageSize"   dc:"每页条数" d:"10" v:"pageSize@min:1|max:200"`
    Pagination bool `json:"pagination" dc:"是否分页" d:"true"`
}

type PageRes struct {
    PageReq
    PageCount  int `json:"pageCount"  dc:"总页数"`
    TotalCount int `json:"totalCount" dc:"总记录数"`
}

// Pack 将分页请求和总记录数组合，填充 PageRes 的完整分页信息。
func (res *PageRes) Pack(req ReqPageFunc, totalCount int) {
    res.TotalCount = totalCount
    res.PageCount = CalPageCount(totalCount, req.GetPerPage())
    res.Page = req.GetPage()
    res.PerPage = req.GetPerPage()
}
```

## 5. DAO Pattern

TRIGGER: DAO, data access object, Columns, Ctx, gf gen dao

Location: `internal/dao/` + `internal/dao/internal/`

Generated by `gf gen dao`.

```go
// internal/dao/internal/foo.go (auto-generated, DO NOT EDIT)
type FooDao struct {
    table   string
    group   string
    columns FooColumns
}

type FooColumns struct {
    Id     string
    Name   string
    Status string
}

var fooColumns = FooColumns{
    Id:     "id",
    Name:   "name",
    Status: "status",
}

func NewFooDao() *FooDao {
    return &FooDao{group: "default", table: "cp_foo", columns: fooColumns}
}

func (dao *FooDao) Ctx(ctx context.Context) *gdb.Model {
    return g.DB(dao.group).Model(dao.table).Safe().Ctx(ctx)
}

// internal/dao/foo.go (extensible)
type fooDao struct {
    *internal.FooDao
}

var Foo = fooDao{internal.NewFooDao()}
```

## 6. ORM Usage Patterns

TRIGGER: ORM, gdb.Model, ScanAndCount, Fields, WhereLike, WherePri, Transaction, IsUnique

### Query (list with pagination)
```go
mod := s.Model(ctx)
mod = mod.Fields(sysin.FooListModel{})
mod = mod.WhereLike(dao.Foo.Columns().Name, "%"+in.Name+"%")
mod = mod.Page(in.Page, in.PerPage)
mod = mod.OrderDesc(dao.Foo.Columns().Id)
mod.ScanAndCount(&list, &totalCount, false)
```

<!-- PATCHED: 2026-05-10 -->
**Pagination MUST use `ScanAndCount`, never manual two-step `Scan` + `Count`.** The traditional approach of calling `Scan` with `Limit/Page` then re-querying `Count` without limit is verbose and error-prone. `ScanAndCount` handles both in one call:

```go
// WRONG — manual two-step pagination
err = orm.Order(in.OrderBy, in.OrderDirection).Limit(in.Offset, in.Limit).Scan(&items)
if err != nil {
    return
}
total, err = orm.Count()

// CORRECT — single ScanAndCount call
if err = dao.UserInfo.Ctx(ctx).Where(do.UserInfo{
        ResourceId: in.ResourceId,
        Status:     in.Statuses,
    }).
    Order(in.OrderBy, in.OrderDirection).
    Page(in.Page, in.PageSize).
    ScanAndCount(&items, &total, false); err != nil {
    return
}
```

The 3rd parameter `useFieldForCount` should always be `false` (executes `COUNT(1)`). Pass `true` only when you need `COUNT(fields_cols)`.

### Insert
```go
s.Model(ctx, &handler.Option{FilterAuth: false}).
    Fields(sysin.FooInsertFields{}).
    Data(in).OmitEmptyData().Insert()
```

### Update
```go
s.Model(ctx).
    Fields(sysin.FooUpdateFields{}).
    WherePri(in.Id).Data(in).Update()
```

### Delete (soft delete)
```go
dao.Foo.Ctx(ctx).WherePri(in.Id).Delete()
```

### Transaction
```go
g.DB().Transaction(ctx, func(ctx context.Context, tx gdb.TX) (err error) {
    return
})
```

<!-- PATCHED: 2026-05-11 -->
**Transaction callback MUST use named return value `(err error)`.** The callback signature must be `func(ctx context.Context, tx gdb.TX) (err error)` — unnamed `error` return type (`func(ctx context.Context, tx gdb.TX) error`) is FORBIDDEN. All transaction-internal error-returning function calls must use inline `if` with bare `return`.

### Unique Check
```go
hgorm.IsUnique(ctx, &dao.Foo, g.Map{dao.Foo.Columns().Name: in.Name}, "名称已存在", in.Id)
```

### Use Struct for WHERE and Data

**Prefer struct over `g.Map` for WHERE conditions and Data insertion.** Struct provides type safety, IDE autocompletion, and cleaner code:

```go
// WRONG — using g.Map with raw string keys
g.DB().Model("work_orders").Where(g.Map{
    "status": "pending",
}).Data(g.Map{
    "title": "test",
}).Insert()

// CORRECT — using struct with dao.Columns() for map keys
var (
    where = g.Map{
        dao.WorkOrders.Columns().Status: "pending",
    }
    data = g.Map{
        dao.WorkOrders.Columns().Title: "test",
    }
)
g.DB().Model(dao.WorkOrders.Table()).Where(where).Data(data).Insert()
```

**When using `g.Map`, keys MUST use `dao.Xxx.Columns().FieldName`** — NEVER raw strings like `"status"`. Raw strings break refactoring and make field dependency tracking impossible.

### OmitEmpty / OmitNil

Use `OmitEmpty()` method instead of `omitempty` struct tag for field filtering. The struct tag is permanently bound to the type and lacks flexibility; `OmitEmpty()` can be selectively applied per operation:

```go
// Recommended: OmitEmpty method
Data(in).OmitEmptyData().Insert()

// Not recommended: omitempty struct tag
type Foo struct {
    Name string `json:"name" orm:"name"`  // do NOT add omitempty here
}
```

- `OmitEmpty()` — filters both WHERE and Data fields where value is zero value (`""`, `0`, `nil`)
- `OmitEmptyWhere()` / `OmitEmptyData()` — filter only WHERE or only Data
- `OmitNil()` — filters only `nil` values, keeps `""` and `0` (useful when you need to update fields to empty string or zero)

> 编码风格（变量声明、命名返回值、godoc 注释、dc 标签等）见 `gfstack-style`。Logic 层中 Model 的调用方式和事务处理见 `gfstack-logic`。
